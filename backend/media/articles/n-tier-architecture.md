## N-Tier Architecture (incompleto)

Antes de publicar esto me indicaron que tenía que definir la arquitectura del software para un programa del trabajo, por lo que me preguntaron qué arquitectura iba a utilizar. Y me mencionaron esta arquitectura (de tantas que hay y se parecen), apenas estoy investigando, pero de momento hay dos términos que es importante definir:

**Layer:** Se traduce como una capa o módulo lógico del software. Una aplicación puede dividirse de forma interna en distintos capas (o módulos) que realizan una función o servicio específico para aislar responsabilidades entre las distintas partes del sistema.

**Tier:** Se traduce como unidad de despliegue. En ocasiones, un aplicación necesita dividirse en partes que se despliegan, administrar, o escalan de manera independiente, pero que trabajan en conjunto. Cada una de estas partes podría denominarse una unidad (tier) y pueden contener una o varias capaz lógicas.
